Overview

What is Mesa?

Mesa is an open-source Python framework for agent-based modeling (ABM) — a simulation approach in which individual agents follow defined rules and system-level behavior emerges from their interactions.

Rather than describing a system through aggregate equations, ABM takes a bottom-up perspective: each agent perceives its local environment and responds according to its own logic. Collective patterns emerge from these individual interactions.

Mesa provides a structured Python API for defining agents and models, running batch experiments, collecting time-series data, and rendering results — all within the standard Python ecosystem.

Core Concepts

Building Blocks

Every Mesa simulation is composed of four core abstractions. Understanding these is sufficient to build and run a model.

01
🤖

Agent

The fundamental unit of a simulation. An Agent is any entity that perceives its environment and takes actions. Agents are Python classes that subclass mesa.Agent. 

class MyAgent(mesa.Agent):
    def step(self):
        # agent behavior defined here
        ...
02
🌐

Model

The top-level container for a simulation. A Model initializes the environment, creates agents, and holds any shared state. It subclasses mesa.Model. 

class MyModel(mesa.Model):
    def __init__(self):
        super().__init__()
        # initialize agents and environment
03
📋

Scheduler

Determines the order in which agents act each step. Mesa provides RandomActivation, SimultaneousActivation, and StagedActivation out of the box. 

# agents act in a random order each step
self.schedule = RandomActivation(self)
self.schedule.add(agent)
04
⏱

Step

One discrete unit of simulated time. Calling model.step() advances the scheduler, which invokes each agent's step() method in turn. Repeat for as many steps as the experiment requires. 

model = MyModel()
for i in range(100):
    model.step()  # advance one time step

Mesa requires Python 3.10 or newer. The package is available on PyPI.

python -m venv venv

pip install mesa

Verify your installation

import mesa
print(mesa.__version__)  # e.g., 3.1.4

Beginner Example

Your First Mesa Model

A minimal money model: agents randomly transfer $1 to another agent. The complete simulation fits in approximately 30 lines of code. 

import mesa

# ── 1. Define the Agent ──────────────────────────────────────────
class MoneyAgent(mesa.Agent):
    """An agent with a fixed initial endowment."""

    def __init__(self, model):
        super().__init__(model)
        self.wealth = 1          # initial wealth

    def step(self):
        # Skip if the agent has no wealth to transfer
        if self.wealth == 0:
            return

        # Select a random agent and transfer $1
        other = self.random.choice(self.model.agents)
        other.wealth += 1
        self.wealth -= 1


# ── 2. Define the Model ──────────────────────────────────────────
class MoneyModel(mesa.Model):
    """A model of pairwise wealth redistribution."""

    def __init__(self, n=10):
        super().__init__()
        self.n_agents = n

        # Batch-create agents; Mesa attaches them to self.agents
        MoneyAgent.create_agents(self, n)

    def step(self):
        # Activate all agents in a random order each step
        self.agents.shuffle_do("step")


# ── 3. Run the Simulation ────────────────────────────────────────
model = MoneyModel(n=50)

for _ in range(20):
    model.step()

# Inspect the resulting wealth distribution
wealth = [agent.wealth for agent in model.agents]
print(wealth)

MoneyAgent subclasses mesa.Agent. Each instance receives a unique_id and a reference to its parent model automatically. The wealth attribute and step() method are user-defined.

MoneyModel subclasses mesa.Model. MoneyAgent.create_agents(self, n) uses Mesa's batch-creation API to instantiate all agents and register them under self.agents automatically.

Each call to model.step() advances time by one step. self.agents.shuffle_do("step") invokes each agent's step() in a freshly randomized order, avoiding systematic activation bias.

Despite uniform initial conditions, wealth becomes concentrated rapidly. This is a well-known ABM result: Boltzmann-like distributions emerge from random pairwise transfers. Increasing the agent count or number of steps produces a stronger effect.

Simulation Flow

How It Works

A Mesa simulation follows a well-defined execution lifecycle. The steps below describe what occurs internally during a run.

🛠

1 · Initialize the Model

Instantiate your Model subclass. The constructor creates agents, sets up any spatial environment (grids, networks, continuous spaces), and initializes shared state.

↓
🔁

2 · Advance the Clock

Each call to model.step() represents one discrete unit of simulated time. The number of steps to run depends on the requirements of the experiment.

↓
🤖

3 · Agents Execute

The scheduler iterates over each registered agent and calls its step() method. Agents read environment state, make decisions, update positions or attributes, and interact with other agents according to their defined behavior.

↓
📊

4 · Collect Data

Mesa's DataCollector records model-level and agent-level metrics at each step. Data is stored as a pandas DataFrame, available for analysis after the run.

↓
📈

5 · Analyze & Visualize

Explore collected data with pandas, matplotlib, or Mesa's built-in browser visualizer. Use BatchRunner to run parameter sweeps and compare outcomes across multiple runs.

Visualization

Visualizing Your Model

Mesa includes a browser-based visualization server that renders agent state and simulation metrics in real time as the model executes.

🌐

SolaraViz (v3+)

Mesa 3 ships SolaraViz, a reactive visualization layer built on Solara. Components such as make_space_component and make_plot_component render agent grids and live charts in the browser with minimal configuration.

🗺

Space Renderers

Render agents on a grid (MultiGrid / SingleGrid), a continuous space, or a network (NetworkGrid). Agent color, shape, and size can map to any attribute on the agent class.

📉

Live Charts

Use make_plot_component to render time-series charts of any metric tracked by DataCollector. Charts update automatically after each step.

🎛

Parameter Controls

Expose model parameters as interactive sliders in the browser UI. Adjust values such as infection rate or agent count and observe the effect on a running simulation — useful for exploration and teaching. 

from mesa.visualization import SolaraViz, make_space_component, make_plot_component

# Define the visual representation of each agent
def agent_portrayal(agent):
    return {"color": "tab:blue", "size": 25}

# Assemble the visualization page
page = SolaraViz(
    MoneyModel,
    components=[
        make_space_component(agent_portrayal),
        make_plot_component("Gini"),       # plot Gini coefficient over time
    ],
    model_params={"n": 50},
)
page  # render in Jupyter, or serve as a web application

Agent

What is an agent?

Each numbered dot is an agent — an autonomous entity with its own position and behavior rule. In Mesa, this corresponds to class MyAgent(mesa.Agent) with a step() method.

Model

What is a model?

The 10×10 grid is the model — the shared environment all agents inhabit. In Mesa this is a mesa.Model subclass that holds the agent collection, the grid, and shared state.

Step

What is a step?

One step is one discrete tick of simulated time. Every agent acts once per step. Use Step → to advance by one step and observe each agent's movement.

Mesa

Mapping to Mesa code

In a real Mesa model: self.model.grid.move_agent() repositions an agent, self.agents.shuffle_do("step") activates them in random order, and DataCollector records state at each step.
